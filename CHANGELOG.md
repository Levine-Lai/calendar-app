# 观赛日记版本更迭说明

这份文件写给使用用户看，用来说明每个版本更新了什么。以后每次发布或准备新版本，都在这里继续更新。

## 当前版本状态

- 手机端最新已发布 APK：`2.2.1`
- 当前源码准备版本：`2.2.2`（尚未打包）
- APK 文件：`releases/sports-calendar-2.2.1-debug.apk`

## 2.2.2（开发中）

- 新闻窗口改为 100% 全屏，充分使用手机屏幕。
- 新闻卡片改为点击标题下拉/收起英文正文，不再跳转外部浏览器。
- 正文按需从 MLB 官方 AMP 页面读取，只提取纯文本段落并在当前会话缓存。
- 原生请求限制为 MLB 官方 HTTPS 地址，并增加 15 秒超时和 768 KB 大小上限。
- 当前只完成源码与测试，尚未生成 `2.2.2` APK。

## 2.2.1（已发布）

- 新增“多伦多蓝鸟新闻”，可在 App 内查看英文标题、摘要、发布时间和 MLB 原文。
- GitHub Actions 每 15 分钟检查一次蓝鸟官网 RSS，直接生成英文新闻 JSON，不调用翻译 API。
- 新增准实时英文通知开关；启用后通过 Firebase 推送新新闻，点击通知可打开对应新闻。
- 新闻列表支持手动刷新、十分钟本地缓存和断网回退。
- 第一次运行只建立新闻基线，不批量推送已有旧新闻。
- 新闻后端不再要求 Firebase Blaze、Cloud Functions 或 Firestore，公开仓库可零成本运行。
- 增加来源链接、响应大小和字段长度校验；本阶段不需要 DeepSeek Key。
- GitHub 免费新闻任务首次手动运行成功，英文新闻 JSON 已可从公网读取。
- 首个 APK 内置最近 20 条蓝鸟英文新闻，其中最近三个北京时间自然日共 15 条，首次打开即可浏览。
- 已生成包含 Firebase FCM 配置的 `2.2.1` Debug APK。

## 2.1.10（未单独发布）

- 应用图标直接使用此前确定的像素跑步小人。
- 移除图标四角的足球、篮球、棒球和橄榄球，只保留小人与浅蓝背景。
- 页面和日历保持不变。
- 桌面组件所选日期没有比赛时不再显示“今日暂无”等占位内容，比赛区域保持空白。
- 本版修改合并进入 `2.2.1`，未单独生成 APK。

## 2.1.9（已发布）

- 左拉菜单最底部新增“应用更新”组件，显示当前版本并支持手动检查更新。
- 检测到新版时展示版本号、更新说明和下载入口。
- 更新检查同时使用两个远程地址容错；断网、超时、清单错误和下载地址未发布时会给出明确提示。
- 下载入口通过 Android 系统浏览器打开，不在 App 内静默安装。
- 修复部分 MLB 等历史比赛将对象比分显示成 `[object Object]` 的问题；对象格式比分会提取真实数值，异常旧记录会先同步再显示。
- 修复赛程过多时备用存储超出容量却误报“主存储保存失败”的问题，不再向用户展示浏览器英文错误。
- 桌面组件1移除底部三个日期圆点，只保留顶部前一天、后一天按钮。
- 加大并拉开左右日期按钮，刷新时间移动到刷新按钮旁边，比赛列表获得更多纵向空间。

## 2.1.8（已发布）

- 重做桌面组件刷新：后台任务由 Android 可靠调度，失败会自动重试，并显示上次更新时间或更新失败。
- 修复中冠等比赛“有比分就误判已结束”的问题，延期和取消状态也会单独显示。
- 多个桌面组件可以各自停留在不同日期，不再互相改变。
- 本地赛程加入自动备份和损坏恢复，旧版本数据会自动迁移。
- 打开 App 时不再立即加载完整球队列表和大量队徽，减少首屏等待。
- 全部球队更新改为并行限流，等待时间更短，部分接口失败时继续保留原赛程。
- 世界杯赛期和中冠年度配置改为自动计算，减少每年必须发布新版的情况。
- 文件导入增加大小、数量、日期和颜色安全检查。
- 加强网络、图片和 WebView 安全限制，减少超大响应造成的卡顿风险。
- 清理项目依赖和生成文件，加入正式签名发布流程，为后续稳定更新做准备。

## 2.1.7（已发布）

- 日历比赛弹窗刷新时只更新比分和状态，不再重建队徽节点，减少打开、关闭和刷新时的图标闪烁。
- 提前预加载当前月份队徽，弹窗中的队徽改为优先加载，并保留已有的非空队徽资源。
- 启动页与 WebView 背景改为应用的浅米色，减少打开 App 时持续 1—2 秒的白屏感。
- 打开 App 后后台预刷新今天和昨天的比赛；未验证的历史 `0 - 0` 会先显示“正在同步”，避免先展示错误比分再跳变。
- 桌面组件日期切换改为先显示本地数据、再后台刷新，并阻止慢请求覆盖已经切换的新日期。
- 放大组件日期按钮点击区域，底部加入过去、今天、未来三个圆点导航。
- 中超改用 ESPN `chn.1` 赛程与实时比分，旧 CFL 中文球队可在点击“更新”后自动迁移。
- 组件刷新加入中超 ESPN 实时比分兜底，修复正在比赛时比分不更新、刷新无响应的问题。

## 2.1.6（已发布）

- 修复未来比赛被误显示为“进行中”的问题。
- 比赛开始时间还在未来时，网页日历弹窗和桌面组件都会强制显示为未开始状态。
- 开赛时间文本例如 `07:10`、`7:10 PM` 不再被误判成实时比赛状态。
- 优化 App 内队徽图片加载：远程队徽会在本地缓存，下一次打开加载更快。
- App 启动图标改为像素小人，并在四角加入足球、篮球、棒球和橄榄球元素。
- 侧边栏标题框只保留“体育迷日历”五个字。
- 桌面组件1改为按日期切换比赛列表，左右箭头可切到前一天/后一天，组件2同步跟随所选日期。

## 2.1.5（已发布）

## 2.1.4（已发布）

- 扩展所有联赛的赛中/完赛状态识别，减少“未开始”“0-0”“已结束”显示不一致的问题。

- 支持按月份查看体育赛程。

- 支持关键词关注球队。

## 使用提醒

- 当前 APK 是个人测试用 Debug 包，不是应用商店正式签名包。
- 手机已安装版本只有在重新打包并安装新 APK 后才会更新。
- 比分实时性取决于对应联赛数据源是否及时更新；App 会尽量刷新，但不能凭空生成数据源没有提供的真实比分。
