# 2.2.1 蓝鸟英文新闻免费部署步骤

当前方案不使用 Blaze、Cloud Functions、Firestore 或 DeepSeek。公开 GitHub 仓库中的 Actions 每 15 分钟读取一次 MLB 官方 RSS，并集中抓取最近 20 篇英文正文；只在内容变化时更新静态 JSON，并可通过免费的 Firebase Cloud Messaging 推送新文章。

## 架构

1. GitHub Actions 运行 `firebase/functions/update-static-news.js`。
2. 脚本读取 `https://www.mlb.com/bluejays/feeds/news/rss.xml`。
3. 脚本读取每篇 MLB 官方 AMP 页面，只保留正文段落并写入 `bodyEn`。
4. 新闻写入 `public/news/blue-jays.json`，无变化时不产生提交。
5. App 从 GitHub Raw 地址一次读取英文标题和正文。
6. 配置 Firebase 服务账号 Secret 后，新增新闻会推送到 FCM 主题 `toronto_blue_jays_news_en`；推送失败只记录日志，不阻断新闻更新。
7. 修改新闻 worker 并推送时，工作流会用 FCM `validate_only` 检查凭据、权限和 REST 消息格式，不会向手机发送测试通知。

## 第 1 步：允许 Actions 写入仓库

1. 打开 `https://github.com/Levine-Lai/calendar-app`。
2. 进入 `Settings` → `Actions` → `General`。
3. 找到 `Workflow permissions`。
4. 选择 `Read and write permissions`。
5. 点击 `Save`。

工作流只会写入 `public/news/blue-jays.json`。

## 第 2 步：生成 Firebase 服务账号

1. 打开 Firebase Console 中的项目设置。
2. 进入“服务账号”页签。
3. 找到“Firebase Admin SDK”。
4. 点击“生成新的私钥”并确认，电脑会下载一份 JSON。

这份 JSON 可以代表后端发送 FCM，属于敏感凭据，不能放入项目、聊天、网盘或公开仓库。

## 第 3 步：保存为 GitHub Secret

1. GitHub 仓库进入 `Settings` → `Secrets and variables` → `Actions`。
2. 点击 `New repository secret`。
3. Name 填写：`FIREBASE_SERVICE_ACCOUNT_JSON`。
4. Secret 粘贴刚下载 JSON 的全部内容，包括最外层 `{}`。
5. 保存后删除下载目录中的明文私钥文件，Firebase 项目中随时可以撤销该密钥。

## 第 4 步：发布代码

将 2.2.1 源码提交并推送到 GitHub `main` 分支。工作流文件为：

`.github/workflows/blue-jays-news.yml`

工作流在默认分支上按计划运行；修改工作流或 `firebase/functions/` 新闻脚本并推送时也会立即运行一次，便于发布后验证。机器人只提交新闻 JSON，不会递归触发。

## 第 5 步：手动运行第一次

1. GitHub 仓库进入 `Actions`。
2. 左侧选择 `Update Blue Jays news`。
3. 点击 `Run workflow`，分支选择 `main`。
4. 等待任务变成绿色。

首次运行用于验证权限和新闻文件，不会把已有的 20 条旧新闻推送到手机。

## 第 6 步：验证静态新闻

打开：

`https://raw.githubusercontent.com/Levine-Lai/calendar-app/main/public/news/blue-jays.json`

能够看到 `teamId`、`updatedAt`、`items` 和每篇文章的 `bodyEn`，说明免费新闻 API 已可用。App 已在 `public/team-news-config.js` 中使用这个地址。

## 第 7 步：验证推送

1. 打包并安装包含 `google-services.json` 的 2.2.1 APK。
2. 打开 App 左侧栏，启用“准实时推送”并允许系统通知。
3. 等 MLB 官方 RSS 发布下一条新文章，Actions 会在约 5 至 20 分钟内发现并推送。

App 端需要保持“准实时推送”开关开启。vivo 手机还应允许通知、自启动，并将“观赛日记”的电池策略设为不限制，否则系统可能延迟后台通知。

## 注意

- 公开仓库的标准 GitHub Actions 免费。
- GitHub 定时任务可能延迟，不保证精确到分钟。
- 公开仓库连续 60 天没有仓库活动时，定时工作流可能自动暂停，需要在 Actions 页面重新启用。
- 已创建的 Firestore 可以保留，但本方案不会读写它，也无需升级 Blaze。
- `android/app/google-services.json` 继续保留，它用于手机连接 FCM；Firebase 服务账号 JSON 只能放 GitHub Secret。
