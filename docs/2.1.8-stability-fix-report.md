# 观赛日记 2.1.8 稳定性修复报告

## 修复目标

本轮针对 2.1.7 全项目审查发现的 19 项问题进行稳定性重构。重点不是增加新联赛或新界面，而是提高比分准确性、组件刷新成功率、数据安全、启动速度和后续维护效率。

## 逐项修复结果

| 编号 | 原问题 | 修复方式 | 验证证据 |
| --- | --- | --- | --- |
| 1 | 组件广播结束后普通线程可能被 Android 终止 | 取消组件 Provider 中的普通线程执行器；即时刷新统一进入唯一 WorkManager 任务，组件先显示本地数据 | Provider 中不再存在比分刷新线程；Android 编译与单元测试通过 |
| 2 | 网络失败仍返回成功，系统不会重试 | 每个数据源记录请求成功/失败；Worker 最多重试 3 次，最终失败会记录原因 | `WidgetRefreshWorker` 根据刷新结果返回 success/retry/failure |
| 3 | 中冠有比分就被当成完赛 | 中国足协比赛优先读取状态字段；缺少状态时结合开赛时间判断，开赛后 4 小时内不会因比分直接完赛 | 新增“赛中已有比分仍为直播”Android 测试 |
| 4 | 本地 JSON 损坏会删除全部赛程 | 新增 IndexedDB 存储层，保留最近两份快照；旧 v5 数据和应急备份可恢复，损坏记录不再直接删除 | 新增损坏主记录恢复测试 |
| 5 | 世界杯和中冠赛季写死 | 世界杯自动计算下一个四年赛期；2026 继续固定 48 队，后续赛期从 ESPN 获取球队；中冠赛事 ID 按年度生成 | 2026 世界杯、中冠接口实际返回成功 |
| 6 | 启动同时加载球队、比分和大量队徽 | 球队列表延迟到打开侧栏后加载；比分刷新放到浏览器空闲阶段；队徽预热从 96 张降至 24 张并限制 4 并发 | Web 语法、构建和测试通过 |
| 7 | 全部球队更新串行且失败等待过长 | 联赛最多 3 并发、球队最多 4 并发；请求改为 2 次、10 秒超时并加入退避；显示联赛更新进度 | 更新流程静态检查与数据源连通测试通过 |
| 8 | 状态文本匹配过宽，延期/取消状态错误 | 建立 scheduled/live/finished/postponed/canceled 五态模型；移除任意单引号、任意 half 等宽泛判断 | Web 与 Android 均新增状态边界测试 |
| 9 | 用户无法判断数据是否最新 | Web 保存上次尝试、成功和错误；组件顶部显示更新时间或更新失败 | 布局资源与 Worker 状态写入编译通过 |
| 10 | JSON/图片无硬上限，可能卡顿或内存溢出 | 原生 JSON 限制 5 MB；图片限制 2 MB、1600 万像素并按尺寸采样后解码；仅允许 HTTPS | Android 编译和 Lint 通过 |
| 11 | 全部赛程反复写入一个大字符串 | Web 主存储改为 IndexedDB 的 meta/events 分离；组件数据改为私有原子文件，最多 5000 场/10 MB | 存储恢复测试、原生编译通过 |
| 12 | 多个组件共用一个日期 | 日期偏移改为按 appWidgetId 保存；列表缓存和数据服务也按组件 ID 隔离 | 服务和 Provider 编译通过 |
| 13 | FileProvider 暴露整个外部存储范围 | 删除未使用 FileProvider 和 `external-path` 配置；关闭应用备份并禁止数据迁移 | Manifest 检查与 Lint 通过 |
| 14 | WebView 允许混合 HTTP 内容 | 改为 `MIXED_CONTENT_NEVER_ALLOW`，外部链接和图片统一限制为 HTTPS | 静态安全搜索通过 |
| 15 | 导出组件 Receiver 可接收内部控制广播 | 新增不导出的 `WidgetActionReceiver`；刷新/切日 PendingIntent 只发给内部 Receiver | Manifest 与 Android 编译通过 |
| 16 | 文件导入没有体积、数量、日期和颜色限制 | 限制 5 MB、10000 场、前后 10 年；文本截断；颜色只接受 3/6 位十六进制；URL 仅 HTTPS | 新增颜色注入测试，Web 测试通过 |
| 17 | 只有 Debug 构建，没有正式签名保护 | 新增正式构建命令；要求四个签名环境变量；Release 开启代码压缩和资源收缩；缺失签名会立即终止 | 发布守卫命令按预期拒绝无签名构建 |
| 18 | 核心文件职责过多，Web/Android 状态逻辑容易漂移 | 拆出 Web 存储、图片缓存；Android 拆出状态、网络、事件存储和内部广播模块；空值合并规则集中到 core | 新模块全部通过语法/编译测试 |
| 19 | 仓库跟踪 `node_modules` 和生成后的 `www` | 新增 `.gitignore`，从 Git 索引移除 2160+ 依赖/生成文件；构建时再生成 `www` | `git ls-files node_modules` 与 `git ls-files www` 均为 0 |

- JavaScript 语法：`app.js`、`calendar-core.js`、`calendar-storage.js`、`calendar-image-cache.js` 全部通过。
- Web 单元测试：12/12 通过。
- Android JVM 测试：15/15 通过。
- Android Lint：构建成功，无 Error/Fatal。
- npm 安全扫描：92 个依赖，0 个已知漏洞。
- Web 构建与 Capacitor 同步：成功，新模块已进入 Android assets。
- 数据源探测：NBA、MLB、NFL、英超、世界杯、中超、中甲/中乙和中冠接口均返回 HTTP 200。
- Git 空白与冲突检查：通过。
- `npm run verify:stability`：19/19 项修复证据全部通过。
- Android Lint 剩余 26 条非阻断警告，主要来自为避免 vivo 再套白框而保留的直接位图启动图标、旧 Capacitor 模板资源和 2x2 组件权重布局；已从审查时的 41 条降至 26 条，无 Error/Fatal。

## 数据兼容与用户影响

- 2.1.7 及更早版本的 Web 本地赛程会从 `sports-fan-calendar:v5` 自动迁移。
- 原组件 SharedPreferences 数据仍可读取，下一次 App 同步后改写为原子文件。
- 多组件升级后日期默认回到今天，此后每个组件独立记忆日期。
- 中冠官方接口不提供明确赛中状态时，使用开赛时间窗口判断；最终准确性仍受官方数据更新时间影响。
